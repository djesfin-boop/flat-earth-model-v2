# 3D Модель Плоской Земли v2.0

Модульная интерактивная 3D модель плоской Земли с азимутальной проекцией.

## Структура проекта

```
flat-earth-model-v2/
├── index.html              # Основной HTML файл
├── css/
│   └── styles.css          # Стили (панели, кнопки, анимации)
├── js/
│   ├── config.js           # Конфигурация и константы
│   ├── astronomy.js        # Астрономические расчёты
│   ├── earth.js            # Создание Земли, координатной сетки, купола
│   ├── celestial.js        # Создание Солнца и Луны
│   ├── scene.js            # Инициализация Three.js сцены
│   ├── controls.js         # UI контролы и обработчики событий
│   └── main.js             # Главный файл запуска
└── azimuthal_map.png       # Азимутальная карта Земли
```

## Модули

### config.js
- Все константы модели (размеры, высоты, настройки)
- Астрономические константы
- Настройки камеры и сцены

### astronomy.js
- Расчёт деклинации Солнца и Луны
- Расчёт позиций светил
- Уравнение времени
- Фазы Луны

### earth.js
- Создание плоскости Земли с текстурой
- Координатная система (меридианы и параллели)
- Тропики Рака и Козерога
- Купол и атмосфера

### celestial.js
- Создание Солнца (со свечением и освещением)
- Создание Луны
- Обновление позиций

### scene.js
- Инициализация Three.js
- Настройка камеры, рендерера, контролов
- Цикл анимации

### controls.js
- Обработка UI событий
- Перемещение и сворачивание панелей
- Обновление информационной панели

### main.js
- Точка входа в приложение
- Запуск всех модулей

## Возможности UI

- **Перемещаемая панель управления** - можно перетаскивать в любое место
- **Сворачивание панелей** - кнопка "+/-" сворачивает панель
- **Полупрозрачность** - свёрнутые панели становятся полупрозрачными

## Использование

1. Откройте `index.html` в браузере
2. Дождитесь загрузки карты
3. Используйте панель управления для настройки

## Технологии

- Three.js r128
- OrbitControls
- Ванильный JavaScript (ES6)
- CSS3

## Примечание

Это рабочий прототип, который будет развиваться. Модульная структура позволяет легко вносить изменения в отдельные компоненты без влияния на остальные части системы.
